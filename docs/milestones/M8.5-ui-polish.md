### Task 8.5.3: Smoothed income rate ($/min)

**Files:** `src/rendering/uiRenderer.ts` (rewrite income tracking)

Replace the current 1-second rolling window with exponential moving average (EMA):

**Current problem:** The income display uses a 60-tick (1-second) window that counts raw fund changes. This causes the $/sec value to jump wildly — it shows the instantaneous burst rate of the last second, not the factory's sustained throughput.

**New approach:**
1. Each tick, compute `instantaneousRate = fundsEarnedThisTick / dt` (raw signal)
2. Smooth with EMA: `smoothedRate += (instantaneousRate - smoothedRate) * (1 - Math.exp(-speed * dt))`
3. Use `speed = 2` for ~1-second convergence (smooth but responsive)
4. Display as `$X.XX/min` (multiply smoothed $/sec by 60) — $/min is more meaningful for automation junkies than $/sec since production cycles are multiple seconds long
5. Show green text when positive, gray when zero

**Bonus stats to display:**
- Total earned: `$${state.stats.totalMoneyEarned}`
- Items sold: `${state.stats.totalItemsSold}`

## Technical Notes

### Income EMA Formula (framerate-independent)

```typescript
// Per tick:
const earned = state.funds - lastFunds;  // raw income this tick
const rawRate = earned / dt;              // instantaneous $/sec
smoothedRate += (rawRate - smoothedRate) * (1 - Math.exp(-SMOOTH_SPEED * dt));
// Display: `$${(smoothedRate * 60).toFixed(1)}/min`
```

With `SMOOTH_SPEED = 2`:
- Responds to changes within ~1 second
- Doesn't spike on individual sales
- Converges to true factory throughput rate

### Minimap Viewport Indicator

```typescript
// Convert camera bounds to minimap coordinates
const scale = minimapWidth / worldPixelWidth;
const viewLeft = (viewport.center.x - screenWidth / 2 / viewport.scale.x) * scale;
const viewTop = (viewport.center.y - screenHeight / 2 / viewport.scale.y) * scale;
const viewW = (screenWidth / viewport.scale.x) * scale;
const viewH = (screenHeight / viewport.scale.y) * scale;
```
